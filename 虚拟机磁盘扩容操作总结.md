# 虚拟机磁盘扩容操作总结

## 📋 问题背景

- **虚拟机平台**: VMware Workstation
- **操作系统**: CentOS 7
- **紧急情况**: `/mnt/docker192` 目录使用率达到 95%，仅剩 11GB 空间
- **数据重要性**: 存储 Docker 服务和 Hadoop 集群数据，无法备份



## 📊 系统概览

### 磁盘空间使用情况 (`df -h`)

| 挂载点                         | 容量     | 已用     | 可用    | 使用率  | 说明                       |
| :----------------------------- | :------- | :------- | :------ | :------ | :------------------------- |
| `/dev/mapper/centos_clc2-root` | 50G      | 37G      | 14G     | 73%     | 系统根分区，空间紧张       |
| `/dev/sda1`                    | 1014M    | 350M     | 665M    | 35%     | 启动分区，正常             |
| `/dev/mapper/centos_clc2-home` | 62G      | 666M     | 61G     | 2%      | 用户目录，空间充足         |
| `/dev/sdb`                     | **197G** | **177G** | **11G** | **95%** | **Docker数据盘，紧急状态** |

## 🔍 详细分析

### 1. 紧急问题识别

- **`/dev/sdb` 使用率**: 95% ⚠️ **严重告警**
- **剩余可用空间**: 仅 11GB
- **数据重要性**: Hadoop集群和Docker服务数据存储

### 2. 磁盘结构分析 (`lsblk`)

text

复制下载

```
sda (120G)                # 系统盘
├─sda1 (1G) → /boot
└─sda2 (119G) → LVM
    ├─centos_clc2-root (50G) → /
    ├─centos_clc2-swap (8G) → [SWAP]
    └─centos_clc2-home (61G) → /home

sdb (200G) → /mnt/docker192  # Docker数据盘（紧急！）
```



### 3. 分区详情 (`fdisk -l`)

- **`/dev/sda`**: 128.8GB 系统盘，使用LVM管理
- **`/dev/sdb`**: 214.7GB (200G) 独立数据盘
  - **无分区表**，整盘格式化为ext4文件系统
  - 直接挂载到 `/mnt/docker192`

### 4. 文件系统信息 (`blkid`)

- **设备**: `/dev/sdb`
- **文件系统**: **ext4**
- **标签**: `docker2`
- **UUID**: `32a3731f-66ec-4443-a462-30d301d6ead6`



## 🎯 扩容目标

将 `/dev/sdb` 磁盘从 200GB 扩容到 400GB，解决空间不足问题



## 🔧 操作步骤

### 阶段一：准备工作

1. **停止 Docker 服务**

   ```sh
   sudo systemctl stop docker.socket
   sudo systemctl stop docker
   sudo systemctl mask docker.socket
   
   # 实际操作
   [root@cd92 ~]# sudo systemctl stop docker.socket
   [root@cd92 ~]# sudo systemctl stop docker
   [root@cd92 ~]# docker ps -a
   Cannot connect to the Docker daemon at unix:///var/run/docker.sock. Is the docker daemon running?
   ```

   

2. **确认服务完全停止**

   ```shell
   sudo systemctl status docker      # 确认状态为 inactive
   sudo ps aux | grep docker        # 确认无残留进程
   sudo lsof +f -- /mnt/docker192   # 确认无文件占用
   ```

   

3. **数据安全同步**

   ```shell
   sudo sync
   sudo sync
   sudo sync
   ```

   

### 阶段二：VMware 层操作

1. **删除虚拟机快照**
   - 进入 VMware 快照管理器
   - 删除无名快照（单个简单快照）
2. **扩展虚拟磁盘**
   - 虚拟机设置 → 硬盘 2 (SCSI)
   - 点击"扩展"按钮
   - 从 200GB 扩展到 400GB
   - 确认扩展成功

### 阶段三：系统层操作

1. **重新扫描磁盘**

   ```shell
   sudo echo 1 > /sys/class/block/sdb/device/rescan
   ```

   

2. **验证新容量识别**

   ```shell
   sudo fdisk -l /dev/sdb
   
   # 实际操作
   [root@cd92 ~]# sudo echo 1 > /sys/class/block/sdb/device/rescan
   [root@cd92 ~]# sudo fdisk -l /dev/sdb
   
   磁盘 /dev/sdb：429.5 GB, 429496729600 字节，838860800 个扇区
   Units = 扇区 of 1 * 512 = 512 bytes
   扇区大小(逻辑/物理)：512 字节 / 512 字节
   I/O 大小(最小/最佳)：512 字节 / 512 字节
   ```

   

   **输出确认**: 磁盘容量从 200GB → 429.5GB

3. **在线扩展文件系统**

   ```shell
   sudo resize2fs /dev/sdb
   
   # 实际操作
   [root@cd92 ~]# sudo resize2fs /dev/sdb
   resize2fs 1.42.9 (28-Dec-2013)
   Filesystem at /dev/sdb is mounted on /mnt/docker192; on-line resizing required
   old_desc_blocks = 25, new_desc_blocks = 50
   The filesystem on /dev/sdb is now 104857600 blocks long.
   ```

   

   **输出确认**: 文件系统块数扩展完成

### 阶段四：验证与恢复

1. **验证扩容结果**

   ```shell
   df -h /mnt/docker192
   
   # 实际操作
   [root@cd92 ~]# df -h /mnt/docker192
   文件系统        容量  已用  可用 已用% 挂载点
   /dev/sdb        394G  177G  200G   48% /mnt/docker192
   ```

   

   **扩容结果**:

   - 容量: 197G → 394G
   - 可用空间: 11G → 200G
   - 使用率: 95% → 48%

2. **恢复 Docker 服务**

   ```shell
   sudo systemctl unmask docker.socket
   sudo systemctl start docker
   sudo systemctl status docker
   sudo docker ps -a
   ```

| 总容量   | 197GB  | 394GB  | +197GB   |
| 已用空间 | 177GB  | 177GB  | 保持不变 |

## 💡 关键技术要点

1. **ext4 文件系统优势**
   - 支持在线扩容，无需卸载文件系统
   - `resize2fs` 命令安全可靠
   - 事务日志确保数据一致性
2. **VMware 特性利用**
   - 支持虚拟机开机状态下扩展磁盘
   - 需要先删除快照才能扩展
3. **服务管理技巧**
   - 需要同时停止 `docker.socket` 和 `docker.service`
   - 使用 `mask` 防止自动重启

## ⚠️ 经验总结

1. **监控预警**: 应建立磁盘空间监控，在达到 80% 时提前处理
2. **快照管理**: 定期清理不必要的快照，避免影响磁盘操作
3. **扩容规划**: 建议一次性扩容足够空间，减少频繁操作
4. **数据安全**: 虽然本次无法备份，但正常情况下应有备份方案

## 🚀 后续建议

1. 设置磁盘使用率监控告警
2. 定期清理 Docker 无用镜像和容器
3. 考虑设置日志轮转策略
4. 规划下一次扩容阈值（建议 80%）
