## 阿里云多服务器内网互通完整排查总结

### 一、问题现象

三台同一VPC下的阿里云服务器（172.16.0.43、44、45），需要实现所有端口互相访问。配置了安全组后，出现：

- `ping` 通（ICMP正常）
- `telnet` / `curl` 不通（TCP连接失败）
- 错误提示：`No route to host` 或 `连接拒绝`

------

### 二、核心知识点：三层防火墙体系

阿里云服务器的网络访问控制有三层，**必须全部放行才能通信**：

| 层级       | 组件                                   | 作用                   | 常见误区                           |
| :--------- | :------------------------------------- | :--------------------- | :--------------------------------- |
| **第一层** | 服务器本地防火墙（firewalld/iptables） | 控制本机端口的出入流量 | ❌ 很多人只改安全组，忘了本地防火墙 |
| **第二层** | 阿里云安全组                           | 云平台层面的访问控制   | ❌ 以为配了安全组就万事大吉         |
| **第三层** | 服务监听状态                           | 服务是否正确监听端口   | ❌ 服务只监听了127.0.0.1而非0.0.0.0 |

**流量必须通过所有三层检查**，任何一层拒绝，连接就会失败。

------

### 三、各层详细排查方法

#### 第一层：本地防火墙（firewalld）

```
# 查看已开放的端口
firewall-cmd --list-ports

# 开放端口（永久生效）
firewall-cmd --permanent --add-port=8109/tcp

# 重载配置
firewall-cmd --reload

# 查看所有规则
firewall-cmd --list-all

# 临时关闭防火墙（测试用）
systemctl stop firewalld

# 重新开启
systemctl start firewalld
```



#### 第二层：阿里云安全组

**关键原则：多个安全组的规则是 OR（并集）关系**

- 只要**任意一个**关联的安全组有允许规则，流量就通过
- 所有安全组的规则合并后统一生效

**配置检查：**

1. 确保三台服务器在同一个普通安全组，或各自安全组都有允许规则
2. 普通安全组需确认"组内连通策略"是否为"组内互通"
3. 入方向规则示例：

| 授权策略 | 协议      | 端口      | 访问来源      |
| :------- | :-------- | :-------- | :------------ |
| 允许     | 自定义TCP | 8109/8109 | 172.16.0.0/24 |

```
# 在服务器上查看实例关联的安全组
curl http://100.100.100.200/latest/meta-data/security-groups
```



#### 第三层：服务监听状态

bash

复制下载

```
# 检查端口监听情况
netstat -tlnp | grep 8109

# 或使用ss命令
ss -tlnp | grep 8109

# 期望输出：0.0.0.0:8109（监听所有接口）
# 如果输出：127.0.0.1:8109，则只监听本地，其他服务器无法访问

# 检查服务进程
ps aux | grep node

# 本地测试服务是否正常
curl -v http://localhost:8109
```

### 五、您的实际案例回顾

#### 问题原因

- ✅ 服务监听正确（本地curl能返回500错误，说明服务正常）
- ✅ 安全组配置正确（已允许172.16.0.0/24所有TCP端口）
- ❌ **本地防火墙未开放8109端口**（firewall-cmd --list-ports 输出中只有2377、7946等端口）

#### 解决方案

```
firewall-cmd --permanent --add-port=8109/tcp
firewall-cmd --reload
```



#### 教训

**本地防火墙是最容易被忽略的第一道关卡**。在排查网络问题时，应该按照"本地防火墙 → 安全组 → 服务本身"的顺序，而不是跳过第一层。

### 六、快速排查命令集合

```
# === 在目标服务器上执行 ===

# 1. 检查端口监听
netstat -tlnp | grep <端口>

# 2. 检查本地防火墙
firewall-cmd --list-ports
iptables -L -n | grep <端口>

# 3. 检查安全组（通过元数据）
curl http://100.100.100.200/latest/meta-data/security-groups

# === 在客户端服务器上执行 ===

# 4. 测试连通性
ping <目标IP>
telnet <目标IP> <端口>
nc -zv <目标IP> <端口>
curl -v http://<目标IP>:<端口>

# 5. 追踪路由
tracepath <目标IP>
```

### 七、个人学习收获

通过这次问题排查，我深刻认识到：

1. **不要跳过任何一层**：本地防火墙是最基本也最容易被忽略的检查点
2. **多安全组规则是OR关系**：不是AND，我之前在这点上产生了错误理解
3. **ping通不代表TCP通**：ICMP和TCP是不同的协议，需要分别检查
4. **阿里云安全组规则秒级生效**：不需要重启服务器
5. **普通安全组默认组内互通**：这是最简单的配置方案
