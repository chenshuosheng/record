## 邮件发送间歇性失败问题排查报告

### 一、问题现象

Linux服务器发送邮件时，**有时成功，有时失败**，无固定规律。



### 二、根本原因

DNS轮询（Round Robin）导致的问题：

- `smtp.exmail.qq.com` 域名解析返回了4个IP地址

- 其中**仅1个IP能正常连接465端口**，其余3个均不可达

- 邮件服务器每次发送时随机选择其中一个IP

- 选中可用IP → 成功；选中不可用IP → 失败（连接超时）

  

### 三、排查过程

| 步骤 | 命令                            | 结果               | 结论                         |
| :--- | :------------------------------ | :----------------- | :--------------------------- |
| 1    | `telnet smtp.exmail.qq.com 465` | Connected          | 端口可达                     |
| 2    | `ping smtp.exmail.qq.com`       | 100%丢包           | 正常（SMTP服务器通常禁ping） |
| 3    | `dig smtp.exmail.qq.com`        | 返回4个IP          | DNS轮询                      |
| 4    | 逐IP测试465端口                 | 仅1个IP通，3个不通 | **问题根因**                 |

**IP连通性测试结果：**

| IP地址         | 465端口 | 状态   |
| :------------- | :------ | :----- |
| 157.255.202.11 | ✅ 通    | 可用   |
| 157.148.36.163 | ❌ 不通  | 不可用 |
| 157.148.45.129 | ❌ 不通  | 不可用 |
| 112.90.80.126  | ❌ 不通  | 不可用 |



### 四、解决方案

#### 方案一：固定hosts解析（推荐，立即生效）

```
# 将域名固定解析到唯一可用的IP
echo "157.255.202.11 smtp.exmail.qq.com" >> /etc/hosts

# 验证配置
ping smtp.exmail.qq.com

# 重启邮件服务（如使用Postfix）
systemctl restart postfix
```

**优点**：简单、立即生效、无需修改邮件服务配置



#### 方案二：使用IP作为relayhost（备用方案）

```
# 修改Postfix配置
postconf -e "relayhost = [157.255.202.11]:465"
postconf -e "smtp_use_tls = yes"
postconf -e "smtp_tls_security_level = encrypt"
postconf -e "smtp_sasl_auth_enable = yes"
postconf -e "smtp_sasl_password_maps = hash:/etc/postfix/sasl_passwd"

# 配置认证信息（如需）
echo "157.255.202.11 your-email@exmail.qq.com:your-password" > /etc/postfix/sasl_passwd
postmap /etc/postfix/sasl_passwd
chmod 600 /etc/postfix/sasl_passwd*

# 重启服务
systemctl restart postfix
```



### 五、验证修复

```
# 连续发送多封测试邮件
for i in {1..5}; do
    echo "Test $i at $(date)" | mail -s "Test $i" recipient@example.com
    sleep 1
done

# 检查日志确认全部成功
tail -20 /var/log/maillog | grep "status=sent"
```



### 六、定期检查建议

可用IP可能随时间变化，建议每月执行一次检查：

```
# 批量测试当前可用的IP
for ip in $(dig +short smtp.exmail.qq.com); do
    echo -n "$ip: "
    timeout 3 telnet $ip 465 2>&1 | grep -q Connected && echo "✅ 可用" || echo "❌ 不可用"
done
```



如发现当前固定的IP变为不可用，重复方案一步骤更新 `/etc/hosts` 中的IP。

### 七、关键结论

| 项目               | 结论                                |
| :----------------- | :---------------------------------- |
| 问题类型           | DNS轮询 + 部分IP不可达              |
| 影响范围           | 约75%的发送失败率（4个IP中3个不通） |
| 解决方案           | 固定hosts解析到可用IP               |
| 是否需要改代码     | 否                                  |
| 是否需要联系云厂商 | 否（可选反馈腾讯企业邮）            |
